# Spec: ИИ-консультант по профориентации (встроенный чат)

**Дата:** 2026-06-05
**Статус:** Утверждён к реализации

## Цель

Добавить в веб-приложение профориентации (React + Vite, RU/KZ swipe-игра «Карта
суперсил») встроенный чат с ИИ-консультантом. Консультант автоматически получает
результаты игры пользователя и использует их как контекст для всех ответов,
помогая школьнику осознанно выбрать будущую специальность. После реализации —
деплой на Vercel.

## Контекст: текущее состояние

- Чистый фронтенд: Vite + React 19 + framer-motion. Бэкенда и ключей API нет.
- Скрипты: `dev` (`vite`), `build` (`vite build`), `preview`.
- `src/App.jsx` уже вычисляет:
  - `scores` (6 суперсил: logic, practice, research, organize, create, connect);
  - `rankedPowers` (отсортированные суперсилы с именами/описаниями);
  - `normalizedScores`;
  - `recommendations` — топ-3 специальности с числовым `specialtyScore`;
  - `archetype` (профиль) и `resultHero` (герой);
  - `comboTrail` (активированные комбо).
- `src/data.js` содержит `cards`, `specialties` (с весами `powers` и переводами
  `title`/`program`/`blurb`), `profileArchetypes`, `heroCharacters`, `localeText`.
- **Пробел:** понравившиеся/отклонённые карточки применяются к `scores` в
  `applyOutcome`, но **не сохраняются** как список. Их нужно начать отслеживать.

## Решения (из брейншторминга)

| Вопрос | Решение |
|---|---|
| Движок ИИ | Реальный LLM через бэкенд |
| Провайдер | Claude (Anthropic), `@anthropic-ai/sdk` |
| Модель | `claude-sonnet-4-6` |
| Хостинг бэкенда | Vercel serverless function (`/api`) |
| Стриминг | Да (SSE, плавный вывод по токенам) |
| UI | Выезжающая панель (drawer) на экране результатов |
| Язык ответов | По текущему `locale` игры (RU/KZ) |
| Финал | Деплой на Vercel |

## Архитектура

```
Браузер (React)                      Vercel Serverless              Anthropic API
─────────────                        ─────────────────              ─────────────
ChatDrawer ──POST /api/chat────────▶ api/chat.js ───messages.stream──▶ claude-sonnet-4-6
  • история сообщений                  • хранит ANTHROPIC_API_KEY
  • профиль из игры (context)          • строит system-prompt
  ◀──── SSE-поток токенов ────────────  • стримит ответ обратно
```

Ключ API (`ANTHROPIC_API_KEY`) живёт только в окружении сервера. Фронтенд его
никогда не получает.

## Компоненты

### 1. `src/chat/gameProfile.js` (новый, чистая функция)

`buildGameProfile({ scores, rankedPowers, recommendations, archetype, resultHero,
comboTrail, decisions, cards, localeText, locale })` → единый объект профиля:

- `likedCards` — заголовки карточек, по которым был `like` (на языке `locale`);
- `skippedCards` — заголовки карточек с `skip`;
- `interests` — суперсилы с числовыми значениями и именами (выявленные интересы);
- `archetype` — титул профиля + краткое описание;
- `hero` — итоговый герой;
- `recommendations` — топ-3 специальности, каждая с:
  - `title`, `program`, `blurb` (на языке `locale`);
  - `matchPercent` — процент соответствия;
- `combos` — активированные комбо;
- `locale`.

**Процент соответствия:** нормализуется к лучшей специальности в наборе
(`matchPercent = round(specialtyScore / maxSpecialtyScore * 100)`), лучшая = 100%.
Если у всех `specialtyScore = 0` (граничный случай), все проценты = 0.

Функция чистая и детерминированная → покрывается unit-тестами.

### 2. `api/chat.js` (новый, Vercel serverless function)

- Метод: `POST`. Тело: `{ messages: [{role, content}], profile, locale }`.
- Валидация входа; при отсутствии `ANTHROPIC_API_KEY` → 500 с понятным сообщением.
- Строит **system-prompt**:
  - роль: тёплый, поддерживающий ИИ-консультант по профориентации для школьника;
  - весь `profile` в структурированном текстовом виде (интересы, like/skip,
    архетип, топ-3 специальности с процентами, комбо);
  - правила: отвечать на языке `locale`; опираться на данные игры; давать
    персональные советы; уметь — объяснять «почему рекомендована конкретная
    специальность», сравнивать специальности, рассказывать о связанных профессиях,
    о карьерных перспективах и востребованности, помогать понять сильные стороны,
    предлагать близкие альтернативы, поддерживать естественный диалог с учётом
    истории; не выдумывать факты, при незнании — честно говорить.
- Вызов: `client.messages.stream({ model: "claude-sonnet-4-6", max_tokens: 1024,
  system, messages })`, проксирование SSE-потока токенов в ответ браузеру.
- Обработка ошибок: типизированные исключения SDK (429 → «слишком много
  запросов, попробуйте позже»; 5xx/overloaded → «сервис временно недоступен»).

### 3. `src/chat/useChat.js` (новый хук)

- Состояние: `messages` (история), `isStreaming`, `error`.
- `sendMessage(text)`: добавляет сообщение пользователя, делает `fetch` к
  `/api/chat` с историей + профилем, читает SSE-поток, инкрементально дописывает
  ответ ассистента в `messages`.
- Вся история отправляется на каждый запрос (Claude API без состояния).

### 4. `src/components/ChatDrawer.jsx` + стили (новый)

- Кнопка-триггер **«Спросить ИИ-консультанта»** на экране `results`.
- Drawer через `framer-motion` (в стиле остального приложения): шапка с
  консультантом, лента сообщений (пользователь/ассистент), индикатор набора при
  стриминге, поле ввода с отправкой по Enter.
- 3–4 стартовых чипа-подсказки: «Почему мне рекомендована эта специальность?»,
  «Сравни топ-2 специальности», «Какие профессии связаны?», «Близкие
  альтернативы?».
- Все UI-строки локализованы через `localeText` (RU/KZ).

### 5. `src/App.jsx` (изменения)

- Новое состояние `decisions` (`[{cardId, outcome}]`); сбрасывается в `beginGame`,
  пополняется в `applyOutcome`.
- На экране `results`: кнопка-триггер и рендер `ChatDrawer`, которому передаётся
  результат `buildGameProfile(...)` и `locale`.

### 6. Конфигурация и зависимости

- `package.json`: добавить зависимость `@anthropic-ai/sdk`; dev-зависимость
  `vitest` для unit-тестов; скрипт `test`.
- `.env.example`: `ANTHROPIC_API_KEY=`.
- `.gitignore`: убедиться, что `.env*` (кроме `.env.example`) игнорируется.
- `vercel.json` при необходимости (обычно Vite + `/api` определяются автоматически).

## Тестирование

- **Unit:** `buildGameProfile` — корректность списков like/skip, расчёт
  `matchPercent` (включая граничный случай нулевых очков), выбор языка.
- **Вручную (локально):** через `vercel dev` (Vite-only `npm run dev` не исполняет
  функции в `/api`). Требуется реальный `ANTHROPIC_API_KEY` в окружении.
- **Вручную (прод):** после деплоя — проверить чат на задеплоенном URL.

## Деплой (финальный этап)

1. Установить/проверить Vercel CLI и линковку проекта.
2. Задать `ANTHROPIC_API_KEY` в переменных окружения проекта Vercel
   (Production + Preview).
3. Задеплоить (preview → проверка → production).
4. Проверить работу чата на боевом URL.

## Вне рамок (YAGNI)

- Аутентификация/учётные записи пользователей.
- Хранение истории переписки на сервере (история живёт в state сессии).
- Аналитика/логирование диалогов.
- Управление расходами/квотами сверх стандартных лимитов API.

## Замечания по локальному запуску

`npm run dev` (Vite) обслуживает только фронтенд — запросы к `/api/chat` вернут
404. Для локальной проверки чата использовать `vercel dev`. В проде на Vercel
фронтенд и функция `/api` работают вместе автоматически.
